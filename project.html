Projekt-Dashboard

Übersicht der funktionalen Kernanforderungen für den Pokémon Card Portfolio Tracker. Die App ermöglicht es Sammlern, ihre Karten zu verwalten und den aktuellen Börsenwert in Echtzeit zu verfolgen.

🗂️

Portfolio

Virtuelle Verwaltung physischer Karten mit Nutzerkonten-Zuweisung.

📈

Echtzeit-Kurse

Marktpreise, Historie (Tag, Woche, Monat, Jahr) inkl. Währungsumrechnung.

📷

OCR Scan

Browser-basierter Smartphone-Scan zur automatischen Kartenerkennung.

📊

Excel Im/Export

Massenimport via SheetJS, Template-Validierung und Portfolio-Export.

Kern-Herausforderungen

  • ⚠️

    Datenqualität & API Limits

    Zuverlässige Preisdaten via TCGPlayer/Cardmarket zu erhalten (Rate Limits, Kosten, API Zugang vs. Scraping).

  • 🤖

    Low-Cost OCR & Machine Learning

    Visuelle Erkennung von Karten (Set-Symbol, Nummer) via Kamera im Browser (WebRTC) mit minimalen Serverkosten (z.B. TensorFlow.js/YOLO clientseitig).

  • 💰

    Währungshandling

    Live-Umrechnung und Anzeige der Originalwährungen (USD, EUR, JPY) je nach Preisbörse inkl. historischer Währungskurse.

Markt & Domänenwissen

Analyse der Pokémon-Karten-Taxonomie, Marktdynamiken und Preisquellen zur korrekten Strukturierung des Datenmodells.

Karten-Taxonomie

Die eindeutige Identifikation einer Karte erfordert komplexe Metadaten. Ein reiner Textabgleich reicht nicht aus.

Name & HP Glurak (Charizard), 120 HP
Set / Erweiterung Base Set, Fossil, XY
Nummer 4/102
Sprache EN, JP, DE
Druckvariante Holo, Reverse, 1st Ed.
Zustand / Grade PSA 10, NM, LP
💱

Börsen & Preisquellen

Preisfindung ist fragmentiert. Die Anwendung muss Multi-Währungen und verschiedene Preisindikatoren unterstützen.

  • TCGPlayer (US): Größter Markt, API oft restriktiv. Preise in USD.
  • Cardmarket (EU): Führend in Europa. Bietet gute API/Export. Preise in EUR.
  • PriceCharting: Historische Daten, oft aggregiert. Gut für Vintage.
  • Metriken: Market Price (empfohlen), Mid, Low, Last Sold.
📸

Bilddaten & Zustand (Grading)

Für die geplante OCR-Erkennung sind hochqualitative Bilddatenbanken essenziell. Die pokemontcg.io API bietet Bilder und Metadatenbanken als Open Source an.

Der physikalische Zustand (Near Mint vs. Lightly Played) oder ein offizielles Grading (PSA, BGS, CGC) verändert den Preis um den Faktor 10 bis 100x. Das Datenmodell muss Preise auf Zustands-Ebene tracken.

Tech-Stack & Architektur

Evaluierung der optimalen Technologien für ein Solo-Entwickler-Projekt mit Fokus auf Performance, Kosten-Effizienz (Low-Cost Hosting) und moderner UX.

Frontend Frameworks

⚛️
  • Next.js (Empfohlen): Ideal für SEO, API-Routen integriert, riesiges Ökosystem, perfekte Vercel-Integration (Free Tier).
  • Vue/Nuxt: Sehr gute Alternative, leichtgewichtiger, aber kleineres US-Börsen-Ökosystem.
  • SvelteKit: Höchste Performance, kleinster Bundle-Size. Top für Mobile-OCR-Webapps.

UI & Tabellen

🎨
  • shadcn/ui + Tailwind: Maximale Anpassbarkeit, Financial-Dashboard Look (MUI ist zu schwerfällig).
  • TanStack Table: Headless Table Library. Absoluter Standard für hoch-performante, sortierbare Portfolio-Listen.
  • Chart.js / Recharts: Für Sparklines (Kursverläufe) in den Tabellenzeilen.

Datenbank & ORM

🗄️
  • PostgreSQL (Supabase/Neon): Relationales Modell ist zwingend nötig (User -> Collection -> Cards -> PriceHistory). Document-DBs (MongoDB) skalieren hier bei Preis-Zeitreihen schlecht.
  • Prisma / Drizzle ORM: Typsicherheit, schnelle Migrationen.

Auth & User Management

🔐
  • NextAuth/Auth.js: Standard für Next.js. Kostenlos, unterstützt OAuth.
  • Supabase Auth: Falls DB bei Supabase liegt, perfekte Integration + Row Level Security (DSGVO konform).
  • Clerk: Einfachstes Setup, aber teuer bei Skalierung.

Mobile Kamera & OCR Scan

📷
  • WebRTC / MediaDevices API: Nativer Zugriff auf Smartphone-Kameras direkt im Browser ohne App-Store.
  • Tesseract.js (WASM): On-Device OCR. Keine Serverkosten! Liest Textfelderbanken.
  • TensorFlow.js (YOLO): Vortrainiertes Object-Detection Modell, um das Set-Symbol oder Artwork im Kamerabild zu matchen (Open-Source Datensätze von HuggingFace).

Import / Export

📄
  • SheetJS (xlsx): Die beste Library zum clientseitigen Parsen von Excel-Dateien (Massenimport) ohne Server-Load.
  • Zod: Schema-Validierung für den Excel-Import (Fehlerbehandlung fehlerhafter Reihen).

Architektur-Empfehlung: T3 Stack Adaption

Next.js (App Router) + Tailwind + Supabase (PGSQL + Auth) + Prisma/Drizzle. Gehostet auf Vercel (Free Tier). Preis-Sync via Vercel Cron-Jobs.

Work Packages (Meilensteine)

Phase 1: MVP (Core Data)

Grundlegende Kartenverwaltung, Datenbankmodellierung, Anbindung pokemontcg.io API, rudimentäre Preisanzeige für Einzelkarten.

Phase 2: Portfolio & Dashboard

Nutzerverwaltung (Auth), Sammlungen erstellen, Dashboard mit Gesamtwert, Tabellen-Ansicht (TanStack Table) mit Historie und Farbindikatoren (rot/grün).

Phase 3: Import / Export

Integration von SheetJS. Download von Excel-Templates, Massen-Upload, Zod-Validierung, visuelles Feedback bei fehlerhaften Zeilen.

Phase 4: OCR Scanner

Kamera-Modul im Browser (WebRTC). Clientseitige Bildvorverarbeitung, Text-Erkennung via Tesseract.js WASM, API-Abgleich und Import in Sammlung.

Phase 5: Währungen & Polish

Wechselkurse (USD, EUR, JPY) implementieren, Mobile-Responsiveness optimieren, Caching-Strategien (Redis/CDN) verfeinern.
